After reviewing the OMDB api and examples, I noticed they had their searches broken out into two top-level types. One using IMDB ID's which returns a single item, or via a String which will return many item references. I merged the two variations into one search section with the service detecting what type of request it is (via ID or Title). Below is a brief explanation of the steps taken to complete this project.

Framework:

I chose the Ionic Framework as its a natural showcase for Angularjs. I am familiar with this framework and felt that it's speed of production, ease of use, and simple UI development allowed for simpler production without the loss of quility UI/UX.

Service:

There is a single service that handles the connection to OMDB and the DOM. Uses promises ($q) to handle Asynchronous calls to API.

Controllers:

These are pretty straight forward and handle most of the logic. There is a Highcharts Config setup in the DetailsCtrl that works with highcharts-ng and the directive in the DOM.

App.js:

One parent route with child routes for individual sections etc.
